# KHStrategyLab Thin MA Signal Engine 설계 문서

작성 기준: `Thin` 브랜치  
작성 목적: 소스코드 수정 전에 **TOP20 리소스 제거 + 매수후보 10분봉 MA 신호등 + KRX/NXT 분봉 분리 + 장마감 저장 기준**을 먼저 확정한다.

---

## 1. 결론

현재 방향은 다음과 같이 확정한다.

```text
TOP20
= 조회 화면 전용
= 0B 실시간 등록 제외
= 장중 KRX 실시간 보정 제외
= 장후/NXT 실시간 오버레이 보정 제외

추적중인 매수후보
= 0B 실시간 유지
= 10분봉 MA5/MA20/MA60 표시
= 신호등 표시
= 5분봉 20봉 신고가 매수 트리거 감시
```

즉, 기존에 TOP20 화면 보정에 쓰던 리소스를 매수후보 실시간 판단으로 옮긴다.

---

## 2. 리소스 우선순위

시스템 리소스가 부족해질 경우 우선순위는 아래와 같다.

```text
1순위: 조건00 매수후보 0B / 10분봉 MA 신호 / 5분봉 20봉 신고가
2순위: 보유종목 평가
3순위: TOP20 조회 화면
4순위: TOP20 실시간 보정
```

TOP20 실시간 보정은 제거 대상이다.  
필요하다면 이후 잔고 실시간 UI 갱신도 줄일 수 있다.  
마지막까지 유지할 것은 매수후보 실시간 신호/전략 판단이다.

---

## 3. 조건검색 역할 기준

현재 `Thin` 기준 역할은 다음과 같이 본다.

```text
조건00
= 장마감 후 후보 리스트업 / 저장 / 추적 / 전략 후보

조건01
= 제거 또는 비활성화

TOP20
= 시장 분위기 조회 화면 전용
```

따라서 0B 등록 대상은 다음과 같다.

```text
보유종목 + 조건00 추적후보
```

TOP20 전용 종목은 0B 등록 대상에서 제외한다.

---

## 4. KRX/NXT 분봉 분리 기준

각 후보는 `StrategyMarket` 값을 기준으로 분봉 소스를 분리한다.

```text
KRX 후보
= KRX 10분봉 / KRX 5분봉 기준

NXT 후보
= NXT 10분봉 / NXT 5분봉 기준
```

같은 종목이라도 KRX와 NXT의 체결 흐름이 다를 수 있으므로, 이평선과 5분봉 20봉 신고가 기준도 분리한다.

가이드 AI 답변 반영 기준:

```text
NXT 분봉 조회 코드 예시: 005930_NX
SOR/통합 예시: 005930_AL
KRX 기본 예시: 005930
분봉 조회 TR 예시: ka10080
10분봉 tic_scope: 10
5분봉 tic_scope: 5
```

단, 실제 키움 가이드/테스트로 아래 항목은 반드시 확인한다.

```text
005930_NX 형식이 실제 운영 환경에서 안정적으로 동작하는지
ka10080이 NXT 전용 분봉을 정확히 반환하는지
응답 최신 봉이 미완성 봉인지 확정봉인지
15:30/15:40 조회값 차이
NXT 20:00 이후 최종 분봉 확정 방식
```

## 5. 초기 분봉 데이터 로드 기준

10분봉 60선을 계산하려면 최소 10분봉 60개가 필요하다.

권장 기준:

```text
10분봉: 최근 100개 조회
5분봉: 최근 30개 이상 조회
```

이유:

```text
MA60 계산 여유 확보
미완성 봉 포함 여부 대응
전날/당일 경계 대응
재시작 후 화면 복원
5분봉 20봉 신고가 계산 여유 확보
```

---

## 6. 장중 현재봉 처리 기준

0B 등록으로 이평선 값이 직접 오는 것은 아니다.  
0B는 현재가/체결시간/체결량 등 실시간 체결 데이터를 제공한다.  
이평선은 분봉 데이터와 현재가를 이용해 내부에서 직접 계산한다.

장중 처리 흐름:

```text
장 시작 전 또는 후보 등록 시
→ 최근 10분봉 100개 조회
→ 최근 5분봉 30개 이상 조회
→ 초기 MA5/MA20/MA60 계산
→ 초기 5분봉 High20 계산

장중 0B 수신
→ 현재 진행 중 10분봉 Close를 현재가로 갱신
→ 현재봉 포함 MA5/MA20/MA60 계산
→ 화면 신호등 갱신

5분봉도 동일하게 현재봉을 갱신
→ 현재가가 최근 완성 5분봉 20개 고가를 돌파하는지 확인

10분/5분 정각
→ 현재봉을 확정봉으로 마감
→ 새 현재봉 시작
```

중요:

```text
0B 틱마다 REST 분봉 재조회 금지
0B 틱마다 차트 재구성 금지
0B 틱마다 전체 그리드 리셋 금지
```

## 7. 화면 표시용 신호등 기준

추적중인 매수후보 그리드에는 아래 컬럼을 추가한다.

```text
10분5
10분20
10분60
신호
```

숫자 표시를 함께 두는 이유는 신호등 계산이 실제 이평선 값과 맞는지 확인하기 위함이다.

신호등 기준:

```text
🟢🟢 = MA5 > MA20 > MA60
🟢   = MA5 > MA20 && MA5 > MA60
🔴   = MA60 > MA5 && MA60 > MA20
🟡   = 나머지 전부
```

판정 순서:

```csharp
if (ma5 > ma20 && ma20 > ma60)
{
    signal = "🟢🟢";
}
else if (ma5 > ma20 && ma5 > ma60)
{
    signal = "🟢";
}
else if (ma60 > ma5 && ma60 > ma20)
{
    signal = "🔴";
}
else
{
    signal = "🟡";
}
```

주의:

```text
신호등 때문에 그리드 순위를 자동 변경하지 않는다.
화면 리셋을 하지 않는다.
선택행을 유지한다.
기존 행 위치에서 숫자와 신호만 갱신한다.
```

---

## 8. 화면 신호와 전략 신호 구분

화면 신호등은 현재봉 포함 실시간 참고값이다.

```text
화면 신호
= 현재봉 포함
= 실시간으로 움직임
= 매수 준비 상태 확인용
```

전략 매수 확정 기준은 더 안정적으로 둔다.

```text
전략 신호
= 10분봉 완성 기준 배열 확정
= 5분봉 20봉 신고가 돌파
= 기준가 위 확인
= 초기 매수 가능
```

즉, 현재봉 중간에 초록이 떴다고 곧바로 올인하지 않는다.  
화면 신호는 계기판이고, 전략 신호는 확정 조건이다.

---

## 9. 신규 매수 기준

신규 매수 가능 조건은 다음과 같이 단순화한다.

```text
10분봉 신호가 🟢 또는 🟢🟢
+
5분봉 20봉 신고가 돌파
+
현재가가 기준가 위
=
신규 초기 매수 가능
```

코드 개념:

```csharp
bool isGreenSignal = ma5 > ma20 && ma5 > ma60;
bool isFiveMinBreakout = currentPrice > high20_5min;
bool isAboveBasePrice = currentPrice > basePrice;

bool canBuy =
    isGreenSignal &&
    isFiveMinBreakout &&
    isAboveBasePrice;
```

여기서 매수는 올인이 아니라 전략용 초기 진입으로 본다.

---

## 10. 기준가 확인

매수 신호 발생 시 기준가 위인지 확인한다.

```text
기준가 A
= 기준봉 생성 전 10분봉 60선 최종 돌파 당시의 60선 금액
또는 갭 출발이면 당일 시가
```

매수 직전 조건:

```text
현재가 > 기준가
```

기준봉 저가는 손절/구조 붕괴 확인에 더 가깝고,  
신규 매수 직전 필터로는 기준가 위 여부가 더 자연스럽다.

## 11. 장마감 저장 기준

장마감 후에는 마지막 확정값을 저장한다.  
이 값은 다음날 시작 기준값이 된다.

### KRX 후보

```text
15:30 KRX 정규장 마감 기준 저장
15:40 이후 조회값은 비교/검증용
15:40 값을 KRX 정규장 MA 저장값으로 덮어쓰지 않음
```

저장 기준:

```text
KRX_1530_CloseSnapshot
```

검증 기준:

```text
KRX_1540_CheckSnapshot
```

### NXT 후보

```text
NXT 후보는 NXT 거래 종료 기준 저장
20:00 이후 NXT 최종 10분봉 기준으로 MA5/MA20/MA60 고정
```

저장 기준:

```text
NXT_2000_CloseSnapshot
```

## 13. REST 조회의 역할

REST 분봉 조회는 매 틱 사용하는 것이 아니다.

사용 시점:

```text
후보 등록 시 초기 데이터 확보
프로그램 재시작 시 복원
장마감 후 최종 검증/저장
오류 발생 시 보정
```

사용하지 말아야 할 시점:

```text
0B 틱마다
화면 신호등 갱신마다
5분봉 신고가 체크마다
```

## 14. 오류 체크 포인트

다음 항목은 로그와 비교 저장으로 확인한다.

```text
REST 분봉 정렬 방향
REST 최신 봉이 미완성 봉인지 여부
0B 체결시간 파싱 정확도
5분/10분 경계 처리
NXT 코드 _NX 정상 조회 여부
KRX 15:30 값과 15:40 값 차이
NXT 20:00 이후 최종값 확정 여부
0B 누락 또는 재접속 이후 내부봉 복원
현재가 부호 처리
거래정지/단일가/동시호가 구간 처리
```

## 15. 구현 순서

소스코드는 아래 순서로 진행한다.

### 1단계: 문서 확정

현재 문서 기준으로 전략 방향과 시간 기준 확정.

### 2단계: 모델 필드 추가

`StockGridRow` 또는 후보 모델에 MA 표시 필드 추가.

```text
Ma5Text
Ma20Text
Ma60Text
MaSignalText
MaSignalBrush
```

### 3단계: 그리드 컬럼 추가

추적중인 매수후보 그리드에 컬럼 추가.

```text
10분5
10분20
10분60
신호
```

### 4단계: 분봉 캐시 엔진 추가

후보별 KRX/NXT 10분봉/5분봉 캐시 관리.

```text
TenMinuteBars
FiveMinuteBars
CurrentTenMinuteBar
CurrentFiveMinuteBar
High20_5m
Ma5_10m
Ma20_10m
Ma60_10m
```

### 5단계: 0B 수신 연결

조건00 후보의 0B 현재가로 현재봉 갱신.

### 6단계: 화면 신호등 갱신

현재봉 포함 MA로 그리드 표시 업데이트.  
단, 화면 리셋/자동정렬 없음.

### 7단계: 매수전략 변경

기존 `MA10/MA60 이탈→회복→10선접촉` 중심에서  
`MA5/MA20/MA60 배열 + 5분 High20 + 기준가 위` 구조로 변경.

### 8단계: 장마감 스냅샷 저장

KRX 15:30, NXT 20:00 기준 최종 MA 저장.  
15:40 KRX 값은 비교/검증용으로만 저장.

## 16. 현재 확정된 방향

```text
TOP20 실시간 보정 제거
조건00 매수후보 실시간 0B 유지
매수후보 10분봉 MA5/20/60 숫자 표시
신호등 표시
현재봉 포함 화면 표시
봉완성 기준 전략 신호
5분봉 20봉 신고가 매수 트리거
기준가 위 확인
KRX/NXT 분봉 분리
KRX 15:30 저장
KRX 15:40 검증
NXT 20:00 저장
```

NXT _NX 분봉 조회가 실제 운영 환경에서 정확히 되는지

## 18. 최종 한 줄 정의

```text
Thin MA Signal Engine은 TOP20 화면 보정 리소스를 제거하고,
조건00 매수후보의 KRX/NXT 분봉 기반 MA 신호와 5분봉 돌파 판단에 리소스를 집중하는 경량 매수 준비 엔진이다.
```
