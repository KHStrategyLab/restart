# KHStrategyLab User Manual

작성 기준: 2026-05-25 / `Codex` 브랜치

이 문서는 KHStrategyLab를 실행하고 화면을 확인하는 사용 기준 매뉴얼이다. 개발 히스토리는 `KHStrategyLab_Update_Log.md`와 `Archive/` 문서를 본다.

---

## 1. 프로그램 개요

KHStrategyLab는 키움 REST/WebSocket API를 이용해 국내 주식 후보를 추적하고, 보유종목·조건00 후보·TOP20·차트를 한 화면에서 확인하는 WPF 데스크톱 프로그램이다.

현재 주요 화면 역할은 다음과 같다.

```text
조건00 추적 후보 = 전략 후보와 MA 신호 확인
보유종목 = 잔고/손익/손절 기준 확인
TOP20 = 시장 분위기 조회 화면
차트 = 선택 종목 일봉/10분봉 확인
로그 = 시스템 상태와 API 처리 흐름 확인
```

---

## 2. 설정 파일

실제 설정은 실행 폴더의 `config.json`에 둔다.

```text
bin/Debug/net10.0-windows/config.json
```

GitHub에는 실제 설정 파일을 올리지 않는다. 공개 저장소에는 샘플인 `config.example.json`만 둔다.

주요 설정값:

```text
AppKey
SecretKey
TelegramToken
TelegramChatId
ConditionSeq00
ConditionSeq01
Budget
MaxSlots
AutoBuy
LiveOrderEnabled
OneShareLiveOrderTestMode
```

주의:

```text
LiveOrderEnabled=false → 신호/차단 로그만 발생, 실주문 전송 없음
LiveOrderEnabled=true  → 리스크가드 통과 시 주문 레이어로 전달
```

실전 주문 가능성이 있으므로 두 컴퓨터에서 동시에 프로그램을 실행하지 않는다.

---

## 3. 실행 순서

일반 실행 흐름은 다음과 같다.

```text
프로그램 시작
→ config.json 로드
→ 저장된 조건00 추적 후보 복원
→ 키움 REST 토큰 발급
→ 계좌번호 확인
→ kt00005 잔고조회
→ WebSocket 연결 및 LOGIN
→ 장상태/0B 실시간 등록
→ 조건검색 목록 조회
→ 조건00 실시간 요청
→ TOP20 1회 조회 후 주기 갱신
```

앱이 정상 시작되면 로그창에서 로그인, 잔고조회, WebSocket LOGIN, TOP20 갱신 로그를 확인한다.

## 4. 화면 기준

### 조건00 추적 후보

조건00 후보는 저장/추적/전략 후보 화면이다.

표시 내용:

```text
종목명
현재가
거래량
거래대금
등락률
회전율
10분 MA5/20/60
MA 신호
```

MA 신호는 현재 후보의 10분봉 배열 상태를 보여주는 참고 계기판이다. 신호등이 바뀌어도 행 순서를 자동으로 바꾸지 않는다.

### 보유종목

보유종목은 `kt00005` 잔고조회와 0B/NXT 보정값을 사용한다.

```text
현재가
보유수량
손익률
손절가
```

20:00 이후부터 다음 거래일 07:00 전까지는 NXT 마지막 평가가격 보정을 유지한다.

### TOP20

TOP20은 키움 `0198` 실시간 종목조회순위 조회 화면이다.

중요 기준:

```text
TOP20은 조회 화면 전용
TOP20 전용 종목은 0B 실시간 등록 대상 아님
TOP20은 MA 신호등/전략 판단/주문 연결 대상 아님
```

TOP20 갱신은 기존 행 객체를 유지하고 값만 갱신해서 방향키 이동 중 선택 포인트가 튀지 않도록 한다.

## 5. 차트

기본 차트는 삼성전자 일봉으로 시작한다.

```text
기본 종목: 005930 삼성전자
일봉 보관: 최근 180개
일봉 표시: 최근 100개
10분봉 표시: 최근 1000개
```

그리드에서 종목을 선택하면 차트가 해당 종목으로 변경된다.

차트 하단 정보:

```text
현재가
거래량
거래대금
회전율
```

회전율은 `ka10001` 종목정보에서 직접 오면 그 값을 쓰고, 없으면 거래량과 상장주식수로 계산한다. 계산할 수 없으면 `---`로 표시한다.

---

## 6. 종료

닫기 버튼은 기본적으로 프로그램을 트레이로 숨긴다. 완전 종료는 트레이 메뉴의 종료를 사용한다.

완전 종료 시 정리 대상:

```text
DispatcherTimer
WebSocket 구독/클라이언트
트레이 아이콘
HttpClient
```

종료 중에는 WebSocket 재연결이나 LOGIN 재전송을 하지 않는다.

---

## 7. 두 컴퓨터 작업 기준

두 컴퓨터에서 동시에 작업하지 않고 장소만 이동한다면 아래 흐름으로 충분하다.

```text
작업 시작 전: pull
작업 완료 후: commit + push
다른 컴퓨터 이동 후: pull
```

`config.json`, `Storage`, `Logs`, DB/캐시 파일은 Git 대상이 아니다. 두 컴퓨터에 각각 로컬 파일로 둔다.
