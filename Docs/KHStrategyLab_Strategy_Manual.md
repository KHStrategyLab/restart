# KHStrategyLab Strategy Manual

작성 기준: 2026-05-25 / `Codex` 브랜치

이 문서는 현재 전략 판단 기준을 설명한다. 코드 구조는 `KHStrategyLab_Architecture_Guide.md`를 본다.

---

## 1. 역할 기준

현재 기준:

```text
조건00 = 저장/추적/전략 후보
조건01 = 비활성 또는 보조 설정값
TOP20 = 조회 화면 전용
보유종목 = 잔고/평가/자동매도 관리
```

TOP20은 전략 판단, MA 신호등, 0B 실시간 보정, 주문 연결 대상이 아니다.

---

## 2. 후보 저장 기준

조건00 후보는 `_watchCandidates`와 `Storage/CandidateUniverse`에 저장된다.

보관 기준:

```text
FirstSeen 기준 6거래일 보관
7거래일차 삭제
같은 종목 재편입 시 FirstSeen 재갱신 금지
```

후보 상태에는 시장, 기준봉, 분봉 캐시 상태, 전략 단계, 알림 상태 등이 저장된다.

---

## 3. KRX/NXT 시장분리

시장분리는 `ka10100` 조회 결과와 기존 저장값을 기준으로 한다.

확정 기준:

```text
NXT 지원 명확 → NXT
NXT 미지원 명확 → KRX
API 실패/불명확 → 기존 시장값 유지 또는 PENDING
```

금지 기준:

```text
ka10100 실패 → KRX 강제 확정 금지
ka10100 실패 → NxtEnabled=false 강제 금지
NXT 후보에서 KRX 분봉 fallback 금지
PENDING 후보 매수 판단 금지
```

---

## 4. 기준봉

조건00 후보의 기준봉은 KRX 정규장 일봉으로 저장한다.

```text
저장 기준봉 = KRX 정규장 일봉
전일종가 기준가 = PreviousClose
NXT 종목이어도 기준봉은 KRX 기준
NXT 장외 종가는 다음날 갭 흐름 참고값
```

기준봉 저장은 장마감 이후 1회 실행하는 것을 기본으로 한다.

---

## 5. MinuteCache

전략 판단은 후보별 분봉 캐시를 사용한다.

키:

```text
종목코드|KRX
종목코드|NXT
```

시장별 요청:

```text
KRX → 6자리 종목코드
NXT → 종목코드_NX
```

필수 데이터:

```text
10분봉 최소 60개 이상
5분봉 최소 20개 이상
```

캐시가 준비되지 않은 후보는 매수 판단을 차단하고 분봉 로드만 예약한다.

---

## 6. MA 신호등

MA 신호등은 매수신호가 아니라 현재 10분봉 배열 상태를 보여주는 계기판이다.

기준:

```text
강   = MA5 > MA20 > MA60
가능 = MA5 > MA20 && MA5 > MA60
약   = MA60 > MA5 && MA60 > MA20
관망 = 나머지
```

신호등은 후보 순위를 바꾸지 않는다. 기존 행 위치에서 숫자와 신호만 바꾼다.

---

## 7. 조건00 MA 돌파 전략

단순히 현재 MA가 녹색이라고 바로 신호를 내지 않는다. 기준봉 이후 눌림과 회복 흐름을 통과해야 한다.

상태 흐름:

```text
WAIT_PULLBACK
→ BELOW_10M_MA60
→ RECOVERED_10M_MA60
→ GREEN_READY
→ BUY_SIGNAL
```

핵심 조건:

```text
기준봉 이후 눌림 확인
10분봉 60선 아래 이탈
10분봉 60선 회복
10분 MA5가 MA20/MA60 위로 회복
5분봉 20봉 신고가 돌파
현재가 > 전일종가 기준가
```

전략 신호가 발생하면 `LiveOrderSignal`을 만들고 주문 레이어로 넘긴다.

---

## 8. 전일상한가 몸통 눌림 시가회복 전략

전일 기준봉이 강한 종목에서 몸통 눌림 이후 시가 회복 흐름을 본다.

상태 흐름:

```text
WAIT_BODY_PULLBACK
→ ESCAPE_READY
→ BUY_SIGNAL
```

핵심 조건:

```text
전일 종가등락률 28% 이상 후보
당일 시가 확인
몸통 눌림 이후 탈출봉 저장
현재가가 탈출봉 고가 돌파
목표/손절 손익비 확인
```

손절 기준은 탈출봉 저가를 사용한다.

---

## 9. 주문 연결 기준

전략 파일은 매수 후보 신호를 만들고, 실제 주문 가능 여부는 주문 레이어가 판단한다.

```text
LiveOrderEnabled=OFF → 신호/텔레그램/로그만 발생, 주문 전송 없음
LiveOrderEnabled=ON  → 리스크가드 통과 시 주문 전송
```

리스크가드:

```text
시장 확정 여부
주문 가격/수량 정상 여부
보유 중복
주문 진행 중 중복
당일 같은 전략 주문 여부
슬롯 제한
예산 기준 수량 계산
```

KRX 후보는 KRX 주문, NXT 가능 후보는 SOR 주문을 사용한다.

---

## 10. 전략 운영 원칙

```text
전략 판단은 차트 화면 렌더링에 의존하지 않는다.
분봉 캐시 READY 전에는 신호를 내지 않는다.
NXT 실패를 KRX 확정으로 해석하지 않는다.
TOP20은 참고 화면으로만 둔다.
실시간 갱신은 행 객체 값만 바꾸고 선택 포인트를 흔들지 않는다.
주문 레이어는 전략 신호와 별도로 항상 리스크가드를 통과해야 한다.
```
